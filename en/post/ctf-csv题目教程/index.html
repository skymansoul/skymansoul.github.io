CTF CSV Challenge Tutorial: What the Challenge Wants and How to Write the Python Script

CSV challenges in CTFs are usually placed under Misc, data analysis, forensics, or scripting. They look like ordinary spreadsheet files, but the real test is not whether you can use Excel. The real test is whether you can find patterns in a large amount of structured data, extract the useful information, and finally recover the flag.

In one sentence: a CSV challenge wants you to process data with a script instead of reading it line by line with your eyes.

1. What a CSV Challenge Usually Wants You to Do

A CSV file is essentially a text file whose fields are separated by commas, for example:

id,char,index
1,f,0
2,l,1
3,a,2
4,g,3

In CTF challenges, the flag may be split, shuffled, hidden, or encoded inside the table. Common goals include the following.

1.1 Search for the Flag Directly

The simplest case is that the flag is hidden in a certain row, column, or note field. The challenge may give you a very large CSV file so that it is hard to find manually.

What you need to do:

  1. Read the CSV.
  2. Iterate through every row.
  3. Search every field for keywords such as flag{, ctf{, or FLAG{.

1.2 Sort by One Column and Concatenate Characters

Many challenges split the flag into individual characters and then shuffle their order, for example:

pos,ch
3,g
0,f
2,a
1,l

Here, the challenge wants you to sort by pos and then concatenate ch:

flag

The key is to identify the “order column” and the “character column”.

1.3 Filter Valid Rows by Condition

Some CSV files contain a lot of noise, and only rows matching a condition are real. For example:

id,value,valid
1,x,0
2,f,1
3,l,1
4,?,0
5,a,1
6,g,1

The challenge wants you to keep rows where valid=1 and then concatenate value.

The core idea is: filter first, extract next, then concatenate.

1.4 Decode Encoded Fields

The CSV may not store characters directly. It may store ASCII, hex, binary, Base64, or other encodings.

For example:

index,ascii
0,102
1,108
2,97
3,103

Here, 102 108 97 103 corresponds to ASCII characters:

flag

So you need to convert numbers into characters:

chr(102)  # f

1.5 Reconstruct an Image or QR Code from Coordinates

A slightly more advanced CSV challenge may give you coordinates and colors, for example:

x,y,color
0,0,0
1,0,255
0,1,255
1,1,0

This kind of challenge may want you to reconstruct an image, a QR code, or a pixel map from the CSV. You need to:

  1. Read the x and y coordinates.
  2. Draw pixels according to the color values.
  3. Save the result as an image.
  4. Scan the QR code or inspect the image to get the flag.

2. What to Check First After Getting a CSV File

Do not start by writing a complicated script. Do some basic observation first.

2.1 Check the Header

The header often reveals the intended solving path, for example:

index, char, order, ascii, hex, x, y, r, g, b, valid

Different fields usually point to different directions:

Field name Possible meaning
index / pos / order Sorting
char / ch / value Character concatenation
ascii ASCII to character
hex Hex decoding
bin Binary decoding
x / y Coordinate drawing
r / g / b RGB image
valid / flag / is_true Filtering valid data

2.2 Check the Amount of Data

If there are only dozens of rows, you can inspect the pattern manually. If there are thousands or tens of thousands of rows, you must write a script.

import csv

with open("data.csv", "r", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    rows = list(reader)

print("Row count:", len(rows))
print("Header:", reader.fieldnames)
print("First 5 rows:")
for row in rows[:5]:
    print(row)

2.3 Check for Abnormal Values

CTF challenges often hide key information in abnormal values, such as:

  • A column has one unusually long row.
  • A row contains { or }.
  • Some numbers clearly fall in the ASCII range: 32 to 126.
  • Some fields look like hex: 66 6c 61 67.
  • Some fields look like Base64, possibly ending with =.

3. Basic Python CSV Reading

Prefer Python’s standard csv library first. It does not require installing third-party packages.

3.1 Read as Dictionaries

If the CSV has a header, DictReader is the most convenient option.

import csv

filename = "data.csv"

with open(filename, "r", encoding="utf-8", newline="") as f:
    reader = csv.DictReader(f)

    for row in reader:
        print(row)

Suppose the CSV is:

index,char
0,f
1,l

Each row will be read as:

{"index": "0", "char": "f"}

Note: values read from a CSV are strings by default. If you need sorting or calculation, convert them to int manually.

4. Script Template 1: Search for the Flag Directly

Use this when the challenge gives you a huge CSV and you suspect the flag is directly hidden inside it.

import csv
import re

filename = "data.csv"
pattern = re.compile(r"flag\{.*?\}", re.IGNORECASE)

with open(filename, "r", encoding="utf-8", newline="") as f:
    reader = csv.reader(f)

    for line_no, row in enumerate(reader, start=1):
        for col_no, cell in enumerate(row, start=1):
            match = pattern.search(cell)
            if match:
                print(f"[+] Found flag at row {line_no}, column {col_no}:")
                print(match.group())

If the contest uses a flag format other than flag{}, change this line:

pattern = re.compile(r"your_prefix\{.*?\}", re.IGNORECASE)

5. Script Template 2: Sort and Concatenate Characters

Use this when the CSV has columns such as index and char.

import csv

filename = "data.csv"

items = []

with open(filename, "r", encoding="utf-8", newline="") as f:
    reader = csv.DictReader(f)

    for row in reader:
        index = int(row["index"])
        char = row["char"]
        items.append((index, char))

items.sort(key=lambda x: x[0])

flag = "".join(char for index, char in items)

print(flag)

The most common thing to change is the column name. If the challenge uses pos and value, change it to:

index = int(row["pos"])
char = row["value"]

6. Script Template 3: Filter Valid Rows and Concatenate

Use this when the CSV has a column marking whether a row is valid, such as valid, is_flag, or status.

import csv

filename = "data.csv"

result = []

with open(filename, "r", encoding="utf-8", newline="") as f:
    reader = csv.DictReader(f)

    for row in reader:
        if row["valid"] == "1":
            result.append(row["value"])

flag = "".join(result)
print(flag)

If sorting is also required, combine filtering and sorting:

import csv

filename = "data.csv"

items = []

with open(filename, "r", encoding="utf-8", newline="") as f:
    reader = csv.DictReader(f)

    for row in reader:
        if row["valid"] == "1":
            items.append((int(row["index"]), row["value"]))

items.sort(key=lambda x: x[0])

flag = "".join(value for index, value in items)
print(flag)

7. Script Template 4: ASCII to Characters

Use this when a field contains numbers such as 102,108,97,103.

import csv

filename = "data.csv"

chars = []

with open(filename, "r", encoding="utf-8", newline="") as f:
    reader = csv.DictReader(f)

    for row in reader:
        num = int(row["ascii"])
        chars.append(chr(num))

flag = "".join(chars)
print(flag)

If the CSV also contains an order column:

import csv

filename = "data.csv"

items = []

with open(filename, "r", encoding="utf-8", newline="") as f:
    reader = csv.DictReader(f)

    for row in reader:
        index = int(row["index"])
        char = chr(int(row["ascii"]))
        items.append((index, char))

items.sort(key=lambda x: x[0])

flag = "".join(char for index, char in items)
print(flag)

8. Script Template 5: Hex Decoding

The CSV may contain hex values such as 66, 6c, 61, and 67.

import csv

filename = "data.csv"

hex_parts = []

with open(filename, "r", encoding="utf-8", newline="") as f:
    reader = csv.DictReader(f)

    for row in reader:
        hex_parts.append(row["hex"])

hex_string = "".join(hex_parts)
result = bytes.fromhex(hex_string).decode()

print(result)

If each row contains a complete hex string, you can decode it row by row:

text = bytes.fromhex(row["hex"]).decode()

9. Script Template 6: Reconstruct an Image from Coordinates

If the CSV contains x, y, value, or r, g, b, it probably wants you to draw an image.

Install Pillow first:

pip install pillow

Black-and-white image reconstruction:

import csv
from PIL import Image

filename = "data.csv"

points = []

with open(filename, "r", encoding="utf-8", newline="") as f:
    reader = csv.DictReader(f)

    for row in reader:
        x = int(row["x"])
        y = int(row["y"])
        value = int(row["value"])
        points.append((x, y, value))

width = max(x for x, y, value in points) + 1
height = max(y for x, y, value in points) + 1

img = Image.new("L", (width, height), 255)

for x, y, value in points:
    color = 0 if value == 1 else 255
    img.putpixel((x, y), color)

img.save("result.png")
print("[+] Saved result.png")

RGB image reconstruction:

import csv
from PIL import Image

filename = "data.csv"

points = []

with open(filename, "r", encoding="utf-8", newline="") as f:
    reader = csv.DictReader(f)

    for row in reader:
        x = int(row["x"])
        y = int(row["y"])
        r = int(row["r"])
        g = int(row["g"])
        b = int(row["b"])
        points.append((x, y, r, g, b))

width = max(x for x, y, r, g, b in points) + 1
height = max(y for x, y, r, g, b in points) + 1

img = Image.new("RGB", (width, height), (255, 255, 255))

for x, y, r, g, b in points:
    img.putpixel((x, y), (r, g, b))

img.save("result.png")
print("[+] Saved result.png")

After saving the image, scan it directly if it is a QR code. If the orientation is wrong, try rotating, flipping, or enlarging it.

10. A Complete Solving Example

Suppose the challenge gives you data.csv:

id,pos,value,valid
1,3,g,1
2,0,f,1
3,9,x,0
4,1,l,1
5,2,a,1

The challenge gives no extra hints and only says “find the flag”.

First, observe the fields:

  • pos: looks like an order.
  • value: looks like a character.
  • valid: looks like a filtering condition.

So the idea is:

  1. Read the CSV.
  2. Keep only rows where valid=1.
  3. Sort by pos from small to large.
  4. Concatenate value.

Final script:

import csv

filename = "data.csv"
items = []

with open(filename, "r", encoding="utf-8", newline="") as f:
    reader = csv.DictReader(f)

    for row in reader:
        if row["valid"] != "1":
            continue

        pos = int(row["pos"])
        value = row["value"]
        items.append((pos, value))

items.sort(key=lambda item: item[0])

flag = "".join(value for pos, value in items)
print(flag)

Run it:

python solve.py

Output:

flag

In a real contest, the output is usually:

flag{xxxxxxxxxxxxxxxx}

11. General Workflow for Solving CSV Challenges

When you encounter a CSV challenge, follow this process:

  1. Check the file name and description: look for hints such as sorting, coordinates, colors, encoding, logs, or traffic.
  2. Check the header: field names usually point to the intended solving direction.
  3. Print the first few rows: confirm what each column looks like.
  4. Count rows and inspect field ranges: judge whether it may be image coordinates, ASCII, or a time series.
  5. Write the smallest script first: make it read, print, and filter.
  6. Add features step by step: sort, concatenate, decode, and draw.
  7. Search for the flag format: finally use regex to extract flag{...}.

12. Common Pitfalls

12.1 Encoding Errors

If utf-8 fails, try:

open(filename, "r", encoding="gbk", newline="")

Or:

open(filename, "r", encoding="utf-8-sig", newline="")

utf-8-sig can handle CSV files with a BOM.

12.2 Numeric Sorting Turns into String Sorting

Wrong example:

items.sort(key=lambda x: x[0])

If x[0] is a string, the order may become:

1, 10, 11, 2, 3

So convert it to an integer:

index = int(row["index"])

12.3 The Separator Is Not a Comma

Some files are called CSV, but actually use semicolons, tabs, or vertical bars as separators.

reader = csv.DictReader(f, delimiter=";")

Tab-separated:

reader = csv.DictReader(f, delimiter="\t")

12.4 The Header Contains Spaces

Some headers may look like this:

 index , char

This will make row["index"] fail. Print the field names first:

print(reader.fieldnames)

Or strip keys after reading:

row = {k.strip(): v.strip() for k, v in row.items()}
